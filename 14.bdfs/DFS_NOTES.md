## 二、DFS 的两种形态

### 1. 遍历型 DFS（无回溯）

只做「走过去、做标记」，标记是**永久性**的，从不撤销。

```cpp
void dfs(int x, int y) {
    visited[x][y] = true;          // 标记，永不撤销
    for (each neighbor) {
        if (!visited[nx][ny])
            dfs(nx, ny);
    }
}
```

**适用题目：** P1162、P1596、P1101、P2036

> P1101 递归只沿固定方向一直走到底；P2036 是「选/不选」二叉树，递归栈本身承载了状态，不需要手动撤销外部数组。

#### 洪水填充型 DFS（遍历型的典型应用）

```cpp
// P1596 Lake Counting —— DFS 版洪水填充
void dfs(int x, int y) {
    if (x < 0 || x >= n || y < 0 || y >= m) return;
    if (grid[x][y] != 'W') return;
    grid[x][y] = '.';              // 进入时标记，永不撤销
    for (int i = 0; i < 8; i++)
        dfs(x + dx[i], y + dy[i]);
}
```

> DFS 洪水填充：代码极短（4 行核心逻辑），在函数入口判断并标记，天然不会重复访问。

**对比 BFS 版洪水填充**：

| | DFS | BFS |
|---|---|---|
| 代码量 | 更短（4 行核心逻辑） | 略长（需显式管理队列） |
| 内存 | 递归栈 O(连通块大小) | 队列 O(连通块大小) |
| 连通块极大时 | 可能爆栈 | 队列占内存但不会爆栈 |
| 标记时机 | 函数入口标记 | ⚠️ 必须**入队时**立即标记（出队时标会 TLE/MLE） |

> ⚠️ **DFS 标记时机**：DFS 在函数入口判断 + 标记，天然安全。BFS 则必须在入队时标记，这是 BFS 最容易踩的坑。

### 2. 回溯型 DFS（有回溯）

核心：**试 → 改 → 递归 → 撤销 → 试下一个**

```cpp
void dfs(int pos) {
    if (terminal) { record(); return; }
    for (each choice) {
        apply(choice);    // 修改全局状态
        dfs(pos + 1);
        undo(choice);     // 恢复全局状态 ← 这就是回溯
    }
}
```

**适用题目：** P1219、P1605、P1019、P2392、P2404

## 三、如何判断是否需要回溯？

| 标准 | 需要回溯 | 不需要回溯 |
|------|----------|------------|
| **需求** | 枚举**所有**方案 / 计数 | 只做**一次**遍历、标记 |
| **状态** | 一个选择会**阻挡**其他选择 | 标记后永不撤销 |
| **典型操作** | `used[i]++/--`、`path.push/pop`、`visited=true/false` | 只设置 `visited=true` |

**经典问题对照：**

- 洪水填充：一个格子淹了就永远淹了 → **不需要回溯**
- 迷宫计数：这条路径走过了不代表其他路径不能经过 → **需要回溯**
- 八皇后：皇后放完要尝试其他位置 → **需要回溯**

## 四、DFS vs BFS 选择指南（详细版）

### 4.1 一句话决策树

```
需要枚举所有方案 / 计数？     → DFS（回溯）
需要找最短路径 / 最少步数？   → BFS
只是遍历 / 标记连通块？       → 都可以，DFS 代码更短
状态空间特别深（>10⁴）？      → BFS（避免爆栈）
需要按字典序输出？            → DFS（天然有序）
```

### 4.2 逐条详解

#### 规则 1：求所有方案 → DFS

DFS 通过回溯可以自然地枚举所有可能路径，在递归树的叶节点收集答案。

```cpp
// P1605 迷宫 —— 数所有路径
void dfs(int x, int y) {
    if (x == fx && y == fy) { ans++; return; }
    visited[x][y] = true;
    for (int i = 0; i < 4; i++) {
        if (valid(nx, ny) && !visited[nx][ny])
            dfs(nx, ny);
    }
    visited[x][y] = false;  // 回溯
}
```

BFS 无法做到这一点：BFS 节点出队后不会再访问，无法枚举不同路径到达同一节点的所有情况。

#### 规则 2：求最短路径 / 最少步数 → BFS

BFS 的按层扩展保证了**首次到达即最优**。DFS 需要遍历所有路径后取最小值，效率低得多。

| | BFS（P1135 电梯） | DFS（求电梯最少按键） |
|---|---|---|
| 时间复杂度 | O(n) | O(2^n)，每条可能路径都走 |
| 首次到达 | 即最优 | 不保证，需比大小 |
| 代码实现 | 队列 + dist 数组 | 回溯 + 全局 min |

```
假设电梯有 200 层，每层数字交替为 +1/-1：
→ BFS：最多访问 200 个状态，O(n)
→ DFS：可能走 2^200 条路径（不剪枝的话）
```

#### 规则 3：连通块 / 洪水填充 → 都可以

| | DFS | BFS |
|---|---|---|
| 代码量 | 更短（4 行核心逻辑） | 略长（需显式管理队列） |
| 内存 | 递归栈 O(连通块大小) | 队列 O(连通块大小) |
| 连通块极大时 | 可能爆栈 | 队列占内存但不会爆栈 |

> 大部分 OI 题目的连通块大小在安全范围内，DFS 更简洁。P1596 同时有 DFS 和 BFS 两版，适合对比。

#### 规则 4：状态空间很深 → BFS

DFS 递归深度 = 搜索路径长度。如果最坏情况深度 > 10⁴，递归栈可能溢出。BFS 用队列存储，在堆上分配，不受调用栈限制。

#### 规则 5：按字典序输出 → DFS

DFS 的搜索顺序天然可控：只要按字典序尝试邻居，叶节点访问顺序就是字典序。BFS 按层扩展，难以保证顺序。

## 五、常见技巧与坑

### 状态传参 vs 全局状态

```cpp
// 方式A：参数传递（P2036, P2392）
// 优点：自动恢复，无需手动回溯
void dfs(int pos, int sour, int bitter) {
    dfs(pos + 1, sour * s[pos], bitter + b[pos]);  // 选
    dfs(pos + 1, sour, bitter);                      // 不选
}

// 方式B：全局状态 + 手动回溯（P1019, P2404）
// 适用于状态较复杂、不想传太多参数的场景
used[i]++;
dfs(i, new_len);
used[i]--;        // ← 常见 bug：忘了这行！
```

### 剪枝

- **P2404**：`for (int i = last; i <= remain; i++)` — 不降序避免 1+2 和 2+1 重复
- **P2392**：当前脑时间 ≥ 已知最优解时可剪枝
- **P1019**：`ans = max(ans, len)` 在递归开头更新，用全局最优辅助剪枝

### DFS vs BFS 标记时机（重要！）

| | DFS | BFS |
|---|---|---|
| 标记位置 | 函数入口标记 | ⚠️ **入队时**立即标记 |
| 为什么？ | 递归进入即代表访问 | 出队才标记会导致同一节点反复入队 |
| 出错后果 | — | TLE / MLE |

```cpp
// DFS：函数入口标记，天然安全
void dfs(int x, int y) {
    if (grid[x][y] != 'W') return;  // 入口判断
    grid[x][y] = '.';                // 立即标记
    for (...) dfs(nx, ny);           // 递归子节点
}

// BFS：入队时标记，否则同一个节点被反复入队
void bfs(int x, int y) {
    q.push({x, y});
    grid[x][y] = '.';   // ✅ 入队后立刻标记
    while (!q.empty()) {
        auto [cx, cy] = q.front(); q.pop();
        // ❌ 如果在这里才标记 grid[cx][cy] = '.'，会 TLE！
        for (...) {
            if (grid[nx][ny] == 'W') {
                grid[nx][ny] = '.';  // ✅ 入队时标记
                q.push({nx, ny});
            }
        }
    }
}
```

### 递归深度

- P1019：n ≤ 20，每词最多 2 次，最深 ~40 层，安全
- P1219：n ≤ 13，深 13 层，安全
- 一般 DFS 深度 > 10⁴ 时考虑迭代写法或用 BFS
- 递归深度 ≈ 搜索路径长度，并非状态总数。比如 P1605 迷宫最多走 n×m 格，深度由路径长度决定

## 六、一句话总结

| 问题 | 答案 |
|------|------|
| 什么时候用 DFS？ | 枚举所有方案、连通性、回溯试错 |
| 什么时候要回溯？ | 全局状态被修改后，后续兄弟分支需要看到修改**前**的状态 |
| 回溯最容易忘什么？ | 递归前 `used[i]++`，递归后忘了 `used[i]--` |
| 回溯的本质是什么？ | 在递归树的同一层，让每个分支面对**相同的初始状态** |
